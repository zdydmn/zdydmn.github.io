Promise ​

Promise 表示一个异步操作的最终结果，与之进行交互的方式主要是 .then 方法，该方法注册了两个回调函数，用于接收 Promise 的终值或本 Promise 不能执行的原因。

promise的出现解决了什么问题？

点我查看

Promise 是一种在 JavaScript 中用于处理异步操作的对象。它主要解决了以下几个问题：

  1. 解决深度嵌套造成的回调地狱问题，使代码易于理解和维护，让开发者可以更清晰的了解异步处理的逻辑
  2. 为异步操作的错误，提供了统一的处理方式
  3. 使用链式调用，更方便的组合或顺序调用多个异步操作
  4. 清晰地表示异步操作的进行状态（成功、失败、进行中）

Promise A+规范 ​

Promise A+规范是一个开源健全可互操作的JavaScript Promise规范。它规定了以下内容：

  • 不管进行什么操作，都返回一个Promise对象，类似于jquery的链式操作。
  • 一个Promise的当前状态必须为以下三种状态中的一种：等待态（Pending）、执行态（Fulfilled）和拒绝态（Rejected）。

当Promise处于等待状态时，Promsie可以转移到执行态或者拒绝态。但是当Promise处于执行态或者拒绝态时，它的状态不可以转变为其他状态，并且必须拥有一个不可变的终值。

Then方法 ​

同时一个Promise方法必须提供一个 then 方法以访问其当前值、终值和据因。Promise的then接受两个参数：

  1. onFulfilled
  2. onRejected

需要注意的是 onFulfilled 和 onRejected 都是可选参数。如果他们的类型不是函数的话，则其必须被忽略。

js
const promise1 = new Promise((resolve, reject) => {
    // 修改状态为执行态
    resolve(1)
})
const onFulfilled = (res) => {
    return res * 2
}

const onRejected = (err) => {
    console.log(err)
}
const promise2 = promise1.then(onFulfilled, onRejected);   

promise2.then((res) => {
    console.log(res) // 2
})

同时then方法必须返回一个 promise 对象 。

onFulfilled ​

如果 onFulfilled 是一个函数，它具有以下特性：

  • 当 promise 执行结束后其必须被调用，其第一个参数为 promise 的终值
  • 在 promise 执行结束前其不可被调用
  • 其调用次数不可超过一次

onRejected ​

如果 onRejected 是一个函数，它具有以下特性：

  • 当 promise 被拒绝执行后其必须被调用，其第一个参数为 promise 的据因
  • 在 promise 被拒绝执行前其不可被调用
  • 其调用次数不可超过一次

Promsie常用方法 ​

Promise 常用方法如下

  • then方法: Promise状态发生变化后的回调函数
  • catch方法: Promise发生错误时的回调函数
  • finally方法: 无论Promise状态发生什么变化都会执行
  • all方法: 执行一组Promise，如果这组中有一个失败立马停止执行，返回失败值，如果都成功则将所有成功值组合成数组返回
  • race方法: 执行一组Promise，那个先完成则返回那个值
  • allSettled方法:执行一组Promise，无论这组是否成功或者失败都会在这组Promise执行完成后执行
  • any方法: 执行一组Promise，只有全失败状态才是失败，有一项成功就是成功状态
  • reject方法： reject 方法用于将 Promise 对象标记为拒绝状态。常用语处理错误情况
  • resolve方法：resolve 方法用于将 Promise 对象标记为成功状态，并传递一个值

then方法 ​

then方法的作用是，在Promise状态发生变化后的回调函数，会返回一个新的Promise对象。同时上一个then的return值会作为下一个Promise的resolved的状态值。接受两个参数

  1. 第一个参数为resolved的状态回调函数（可选）
  2. 第二个参数是rejected状态的回调函数（可选）
js
const promise = new Promise((resolved) => {
    resolved(2)
})
promise.then((res) => {
    console.log(res); // 2
    return res * 2
}). then((res) => {
    console.log(res); // 4
})

catch方法 ​

catch方法是发生错误时的回调函数。需要注意的是如果想要在Promise中抛出错误，必须要在状态修改之前，因为如果Promise状态发生改变，则这个状态就不会在变化。

js
const promise = new Promise((resolved) => {
    resolved(2)
    throw new Error('test');
})

promise.then(res => {
    console.log(res) // 2
}).catch((err) => {
    console.log(err)
})

如果catch方法进行链式调用，那么第二个catch方法会补货第一个catch方法抛出的错误

js
const promise = new Promise((resolved, reject) => {
    throw new Error('test ---- 1');
})

promise.catch((err) => {
    console.log(err, 1) // Error: test ---- 1
    throw new Error('test ---- 2');
}).cathc(err => {
    console.log(err, 2) // Error: test ---- 1
})

finally方法 ​

finally方法可以指定无论Promise的状态变为什么都会执行的操作

js
Promise.resolve(2).then(res => {
    console.log(res) // 2
    throw new Error('test ---- 1');
}).catch(err => {
    console.log(err) // Error: test ---- 1
}).finally(() => {
    console.log('finally执行') // finally执行
})

all方法 ​

手写Promise.all

all方法用于将多个Promise包装成一个新的Promise，这个新的Promise状态会根据接受的多个Promise状态发生变化。

js
const new_promise = Promise.all([p1, p2, p3])

如上述例子，如果p1，p2，p3的状态都为成功（fulfilled），那么返回的new_promise的状态也为成功，同时会把三个成功状态的返回值包装成一个数组传递给new_promise的回调。但是如果p1，p2，p3中有任何一个状态为失败（rejected），那么new_promise的状态为失败，此时会讲第一个失败状态的返回值传递给new_promise的回调。

race方法 ​

race方法同样用于将多个Promise包装成一个新的Promise，与all方法不同的是，如果这多个Promise中，有一个的状态率先发声了变化，那么race方法就会将这个更新的返回值传递给这个新的Promise的回调。

js
const new_promise = Promise.all([p1, p2, p3])

以这段代码为例，如果p1，p2，p3中第一个进行状态改变的返回值会传递给new_promise的回调函数。

allSettled方法 (ES2020引入) ​

allSettled方法常用于等待一组Promise执行完毕后，再执行某些操作，无论这组Promise的结过是成功还是失败。用于弥补all方法的缺陷，因为all方法如果有一个Promise执行失败就会结束，而不会等待这组Promise执行完毕。

js
const p1 = Promise.resolve(2)
const p2 = Promise.reject(3)
const p3 = Promise.resolve(4)


Promise.allSettled([p1, p2, p3]).then((res) => {
    console.log(res); // [{ status: 'fulfilled', value: 2 },{ status: 'rejected', value: 3 },{ status: 'fulfilled', value: 4 }]
})

any方法 (ES2021引入) ​

any方法同样也是接受一组Promise作为参数，返回一个新的Promise。他会根据这组Promise的状态来绝对新返回Promise的状态，只要这组中有一个Promise状态为成功，那么新的Promise状态就是成功；只有组中所有的Promise的状态都为失败，新的Promise状态才会为失败。
